Getting Started

The math widgets library has a few dependencies:

  • jQuery
  • underscore.js
  • ttm-coffeescript-utilities
  • ttm-coffeescript-math

Downloading Manually

The latest version of the math widget library can be downloaded via the links below: Each dependency mentioned above

  • Minified
    • https://raw.github.com/thinkthroughmath/javascript-calculator/master/dist/javascript-calculator.min.js
    • https://raw.github.com/thinkthroughmath/javascript-calculator/master/dist/javascript-calculator.min.css
  • Expanded
    • https://raw.github.com/thinkthroughmath/javascript-calculator/master/dist/javascript-calculator.js
    • https://raw.github.com/thinkthroughmath/javascript-calculator/master/dist/javascript-calculator.css

Bower Component

The javascript-calculator library is available as a bower component. Bower makes wonderfully easy to include external components in your project.

Using Coffeescript and Sass

The calculator is written in Coffeescript and styled with Sass. If, for some reason, you wish to either customize the calculator code or compile it in your own build pipeline, it should be straightforward to compile both the Coffeescript and the Sass using your preferred methods. The build task defined in Gruntfile.js defines the calculator is build process.

Including in your Website

Once downloaded, include the javascript-calculator dependencies and the javascript-calculator library itself in your site. The javascript-calculator library exports all of its functionality to a global thinkthroughmath object.

Calculator Usage

The calculator widget is meant to emulate a real-world calculator, like so:

To render the calculator within a dom element, use the build_widget method: 


    <div class="location-to-render-calculator javascript-calculator-themed"></div>

    <script>
      $(function(){
        $(".location-to-render-calculator").each(function(){
          thinkthroughmath.widgets.Calculator.build_widget($(this));
        });
      });
    </script>

In the example above, the class "javascript-calculator-themed" is added to a parent element of the rendered calculator. This enables the specific css styling we use. Its likely you will want to use your own styling, so if you do not add that class to a parent element, the calculator will look "themeless" like below:

If the default button ordering doesn't work for you, you can specifiy the ordering like so: 

    <div class="location-to-render-calculator javascript-calculator-themed"></div>

    <script>
      $(".location-to-render-calculator-with-fewer-buttons").each(function(){
        ttm.widgets.Calculator.build_widget($(this), ['0','1','2']);
      });
    </script>

The default rendering includes all available options, and is quivalent to the Javascript array: 

    defaultButtons = [
        "square", "square_root", "exponent", "clear",
        "pi", "lparen", "rparen", "division",
        "7", "8", "9", "multiplication",

These may be reordered or removed.
